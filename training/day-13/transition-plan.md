## 具体的なプロンプトテンプレート（初月に整備）

### 1. 提案資料の構成設計

```
以下の条件で提案資料の構成案を作ってください。

テーマ：HR-AIサービスの新規顧客向け提案
対象：従業員100〜500名の中堅企業の人事部長
目的：無料トライアルの実施合意を得る
枚数：13枚程度

まず構成案（各スライドのタイトル）だけ出してください。
内容はまだ書かないでください。
```

→ 確認後「スライド3の要点を3-5個出してください」と段階的に進める（Day7の学び）

### 2. 競合リサーチ

```
HR Tech領域におけるエンゲージメント管理ツールの主要競合を5社リストアップし、
以下の観点で比較表を作ってください。

比較軸：主力サービス / 強み / 弱み / 価格帯 / ターゲット企業規模

※ 数値データは出典元も記載してください。
```

→ 出力後、数値は必ず公式サイトで裏取り（Day5のハルシネーション体験から）

### 3. メール下書き

```
以下の条件でビジネスメールの下書きを作ってください。

宛先：展示会で名刺交換した人事部長
目的：展示会のお礼と、デモのご案内
トーン：丁寧だが堅すぎない
含めたい内容：
- 展示会でお話しした内容への言及
- 30分のオンラインデモのご提案
- 来週の候補日を3つ提示
```

## 注意事項・リスク管理

### AI出力の確認ルール
- 数値・固有名詞はAIの出力をそのまま使わない。必ず一次情報（公式サイト・社内データ）で確認する
- Day5の教訓：ソフトバンクの数値がGeminiとChatGPTで食い違っていた経験から、複数ツールの出力を鵜呑みにしない

### 機密情報の取り扱い
- 社内の機密データ（顧客情報・未公開の財務データ等）はAIに入力しない
- 公開情報・匿名化されたデータのみ使用する
- 社内のAI利用ガイドラインが策定されたら、それに従う

### 品質管理
- AI出力は「たたき台」であり、最終成果物ではない（Day7の学び：AIはアクセル、人間はハンドル）
- 重要な資料は必ず自分でレビューしてから提出・共有する
- 段階的指示（構成→要点→詳細）を使い、いきなり完成版を求めない

### AI利用制限への対処
- Claudeの制限に引っかかった場合はChatGPTに切り替える（Day9の体験）
- 重要なタスクは午前中の制限枠に余裕があるうちに実施する
- 1つのツールに依存しすぎず、複数ツールを使い分ける
