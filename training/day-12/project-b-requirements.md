# 要件定義書：新入社員オンボーディングダッシュボード
 
---
 
## 1. 背景
 
新入社員は入社直後、やるべきことが多岐にわたる一方で、「今日何をすべきか」「どこまで進んでいるか」を把握しにくい状況に置かれがちである。特にAI Boot Campのような集中研修では、Day1〜Day13の課題・クイズ・提出物が日々変わるため、進捗の「見える化」が自走力を高めるうえで不可欠である。
 
現状は口頭や個別のドキュメントで管理されており、全体の進捗状況を一目で確認できるツールがない。
 
---
 
## 2. ゴール
 
- **今日やることが一目でわかる**: ランディングページを開いた瞬間に「今日のタスク」が表示される
- **進捗の自己把握**: 研修全体のどこにいるかをグラフ・チャートで可視化する
- **達成感の演出**: 完了したタスクをチェックすることで前進している実感を持てる
 
---
 
## 3. スコープ
 
### 作るもの
- ブラウザ完結のフロントエンドアプリ（Next.js + Tailwind CSS）
- 今日の日付から「今日のDay」を自動判定して表示するランディングページ
- Day別タスク一覧と完了チェック機能
- 研修全体の進捗バー（完了Day数 / 全13Day）
- 完了済みタスクのLocalStorage永続化
 
### 作らないもの
- ログイン・認証機能
- バックエンドサーバー・外部データベース
- 複数ユーザーの進捗共有・比較機能
- 管理者向けの一括管理画面
 
---
 
## 4. ターゲットユーザー
 
- **主ターゲット**: AI Boot Campを受講中の新入社員（研修Day1〜Day13を進行中）
- **副ターゲット**: 入社1ヶ月以内の新入社員全般（研修以外のオンボーディングタスクにも転用可能）
 
---
 
## 5. 主要機能
 
### 機能1: 今日のDayカード（ランディング）
- トップに「今日は Day X」を大きく表示（日付から自動計算 or 手動選択）
- 今日のテーマと必須課題をカードで表示
- 「クイズを受ける」「提出する」などのクイックリンクボタン
 
### 機能2: タスクチェックリスト
- 各Dayの必須課題・提出物をリスト化
- チェックボックスで完了 / 未完了をトグル
- 完了状態はLocalStorageに保存し、再訪時も維持
 
### 機能3: 全体進捗バー
- 13日間のうち何Dayが完了済みかを進捗バーで可視化
- 完了済みDayはアイコンを変えて一目で識別できるようにする
 
### 機能4: Day切り替えナビゲーション
- Day1〜Day13をサイドバーまたはタブで切り替え可能
- 各Dayのテーマ・タスク・提出物を確認できる
 
### 機能5: 完了お祝いアニメーション（Nice-to-have）
- 全タスクをチェックした際に「Today Complete!」の演出を表示
- モチベーション維持のUX向上
 
---
 
## 6. 画面構成
 
**ページ数：1ページ（シングルページアプリ）**
 
```
┌─────────────────────────────────────────────┐
│  🚀 Onboarding Dashboard        [Day選択▼]  │  ← ヘッダー
├────────────────┬────────────────────────────┤
│                │  📅 Today: Day 11           │
│  Day1  ✅      │  テーマ: Project A 実装      │
│  Day2  ✅      │  ──────────────────────     │
│  Day3  ✅      │  📋 今日のタスク             │
│  Day4  ✅      │  ☑ Cursorでプロジェクト作成  │
│  Day5  ✅      │  ☑ AIでコード生成            │
│  Day6  ✅      │  ☐ Vercelにデプロイ          │
│  Day7  ✅      │  ☐ リリースノート作成        │
│  Day8  ✅      │  ☐ クイズ100点達成           │
│  Day9  ✅      │                             │
│  Day10 ✅      │  ████████████░░░  9/13 Day  │  ← 進捗バー
│  Day11 🔵      │                             │
│  Day12 ○       │  [📎 クイズを受ける]         │
│  Day13 ○       │  [📤 PRを提出する]           │
│                │                             │
└────────────────┴────────────────────────────┘
       ↑                    ↑
  サイドバー            メインエリア
  (Day一覧)          (当日詳細 + タスク)
```
 
### 各エリアの詳細
 
| エリア | 内容 |
|--------|------|
| ヘッダー | アプリ名・今日の日付・Dayセレクター |
| サイドバー | Day1〜13の一覧。完了済み✅・本日🔵・未完了○ でステータス表示 |
| メインエリア（上） | 今日のDayカード（テーマ・概要） |
| メインエリア（中） | タスクチェックリスト |
| メインエリア（下） | 全体進捗バー＋クイックリンク |
 
---
 
## 7. 非機能要件
 
| 項目 | 要件 |
|------|------|
| 対応デバイス | PCブラウザ優先（Chrome / Edge / Safari）、スマホ対応は任意 |
| データ保存 | LocalStorageのみ（外部サーバー送信なし） |
| パフォーマンス | チェックトグルは即時反映（遅延なし） |
| 実装規模 | 1日で実装・デプロイ完了できる規模 |
 
---
 
## 8. 技術スタック
 
| 項目 | 使用技術 |
|------|---------|
| フレームワーク | Next.js（App Router）|
| スタイリング | Tailwind CSS |
| 状態管理 | React useState + LocalStorage |
| チャート | 不要（CSSで進捗バーを実装） |
| ホスティング | Vercel |
