## 必須課題

### 課題1: 要件定義書を作成する

自分の業務で実際に使えるWebツールの要件を定義する。

**テーマの選び方（自由。以下は例）:**
- 商談管理ダッシュボード
- 展示会リード管理ツール
- コンテンツ作成チェックリスト
- 競合比較表の自動生成ツール
- 社内FAQ検索ツール
- 議事録テンプレート生成ツール

**Step 1: アイデアをAIと壁打ちする**

Claude に以下のように相談する:

```
私はGTM（セールス/マーケティング）の仕事をしています。
日々の業務で「こんなツールがあったら便利なのに」と思うものを作りたいです。

以下の制約の中で、実現可能なWebツールのアイデアを5つ提案してください:
- 1-2ページ程度のシンプルなWebアプリ
- 非エンジニアがAIの助けを借りて1日で作れる規模
- 自分の業務で実際に使えるもの
- データベースは不要（ブラウザ上で完結）
```

**Step 2: テーマを1つ選び、要件定義書を作成する**

```
[選んだテーマ] の要件定義書を作成してください。
以下の構成で:

1. 背景（なぜこのツールが必要か）
2. ゴール（このツールで何を実現するか）
3. スコープ（作るもの / 作らないもの）
4. ターゲットユーザー（誰が使うか）
5. 主要機能（箇条書きで3-5個）
6. 画面構成（何ページ必要か、各ページの概要）
7. 非機能要件（スマホ対応の要否、パフォーマンス等）
```

**Step 3: 自分の業務知識を加えて修正する**

AI の出力を確認し、以下を修正・追記する:
- 実際の業務フローに合っているか
- 機能の優先順位は正しいか
- 「あると嬉しい機能」と「必須の機能」を区別する

成果物: `training/day-10/project-a-requirements.md`

### 課題2: V0 / Google Stitch でモックアップを作成する

モックアップ = 完成イメージの見本。実際に動くものではなく「こういう画面にしたい」というデザイン案。

**パターンA: V0 を使う（推奨）**

1. https://v0.dev を開く
2. Vercel アカウントでログイン（GitHub アカウント連携）
3. テキスト入力欄に、ツールのイメージを文章で入力する:

```
以下のWebアプリのUIを作ってください:

[要件定義書の「画面構成」と「主要機能」を貼り付ける]

デザイン要件:
- モダンでシンプルなデザイン
- レスポンシブ対応（PCとスマホ両方で見やすい）
- 日本語のUIテキスト
```

4. V0 がUIデザインとReactコードを生成する
5. プレビューで確認し、修正指示を追加で入力する（例: 「ボタンをもっと大きく」「色をブルー系に」）
6. 気に入ったデザインをスクリーンショットで保存する

**パターンB: Google Stitch を使う**

1. https://stitch.google.com を開く
2. Google アカウントでログイン
3. テキストでUIのイメージを入力する
4. 生成されたモックアップを確認・調整する

**パターンC: 手書き + AI でモックアップを作る**

1. 紙にUIの手書きスケッチを描く
2. スマホで写真を撮る
3. Claude に写真をアップロードして「このスケッチをもとにUIの詳細を説明して」と指示
4. 説明をもとに V0 や Stitch でモックアップを生成する

成果物: `training/day-10/project-a-mock.md`（モックアップのスクリーンショットまたは説明を記載）

## Git操作 — AI にやってもらおう

Day4〜9 では Git コマンドを手動で入力してきた。**Day10 からは、Git 操作も AI に依頼してみよう。**

これまで覚えたブランチ作成・add・commit・push の流れは同じだが、コマンドを自分で打つ代わりに AI に指示する。

### GitHub CLI（gh コマンド）のセットアップ

GitHub CLI をインストールすると、**PR の作成やコメントもターミナルから実行できる**ようになる。つまり AI に PR 作成まで一気にやってもらえる。

**インストール:**

```bash
brew install gh
```

**ログイン:**

```bash
gh auth login
```

対話形式で質問されるので以下のように回答する:
- `GitHub.com` を選択
- `SSH` を選択
- 既存の SSH キーを使う（Day4 で設定済み）
- `Login with a web browser` を選択 → ブラウザが開くので GitHub にログインして認証する

**動作確認:**

```bash
gh auth status
```

`Logged in to github.com` と表示されれば成功。

> **うまくいかない場合**: エラーメッセージをそのまま AI に貼り付けて聞こう。

---

### Git + PR 作成を AI に依頼する

GitHub CLI が使えるようになったので、ブランチ作成から PR 作成まですべて AI に任せられる。

**Cursor の AI チャット（Cmd+L）で以下のように依頼する:**

```
研修リポジトリ（~/Desktop/spec-ai-training）で以下の操作をしてください:
1. main ブランチの最新を pull
2. training/day-10-project-a-design という新しいブランチを作成
3. training/day-10/ フォルダの変更をすべて add
4. コミットメッセージ「docs: add Day10 Project A requirements and mockup」でコミット
5. リモートに push
6. gh コマンドで PR を作成（タイトル: 「Day10: Project A 要件定義 + モックアップ」）
```

> **ポイント**: AI がターミナルでコマンドを実行してくれる。何をしているか確認しながら進めよう。手動でやっていた操作と同じことを AI が代行しているだけだとわかるはず。
>
> PR 作成後、GitHub 上で **base repository** が自分のリポジトリになっていることを確認すること。
