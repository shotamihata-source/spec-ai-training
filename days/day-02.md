# Day 2 — 4/2 (木): Claude製品リサーチ

## 今日のゴール

Claude の3つの製品（Chat, Cowork, Code）の違いを理解し、それぞれの特徴・使い分け・ユースケースを資料にまとめる。

---

## 必須課題

### 課題1: Claude を使って3製品の違いをリサーチする

Claude の Web アプリ・デスクトップアプリ、どちらを使用しても構いません。
**ただし、Claude のデスクトップアプリを必ずダウンロードしておいてください。** → https://claude.ai/download

**Step 1: Claude のWebアプリでリサーチを開始する**

1. https://claude.ai を開く（デスクトップアプリでもOK）
2. 新しいチャットを開く
3. 以下のプロンプトを入力する:

```
Claude Chat、Claude Cowork、Claude Code の3つの製品について教えてください。
それぞれの以下の点を詳しく説明してください:
1. 製品の概要（何ができるか）
2. 主な対象ユーザー
3. 具体的なユースケース3つ以上
4. 他の2製品との違い
5. 使い分けの判断基準
```

4. 出力を読み、わからない点があれば追加で質問する

**Step 2: 実際に触って確認する（※可能であれば）**

1. **Claude Chat**: https://claude.ai のチャット画面で質問を投げる → これが Chat
2. **Claude Cowork**: Claude のチャットで「Projects」機能を使う、またはドキュメント連携を試す
3. **Claude Code**: ターミナルから使うCLIツール。インストール方法をClaudeに聞く:

```
Claude Code のインストール方法と基本的な使い方を教えてください。
```

**まだインストールしなくてOK。** 概要を理解することが目的。実際に触れればベストだが、概要の把握が優先。

### 課題3: 成果物を Google Docs で作成し、Google Drive に格納する

**重要: 手動でファイル作成・アップロードはしない。AIやツールを使うこと。作成した資料を一度手動でダウンロードするのはOK。**

**資料作成や格納は Claude を使っても他の AI やツールを使ってもOK。** いろいろなパターンを試してみてください。

格納先: https://drive.google.com/drive/u/0/folders/1_fnoo33Kf6qNzuE6-2dB_rqac2uVNnws

**方法A: Claude にGoogle Docs形式のテキストを生成させる**

1. Claude に以下のように指示する:

```
上記のリサーチ結果を、Google Docs にコピペできる形式で整理してください。
見出し、箇条書き、表を使って読みやすくしてください。
```

2. 出力をコピーする
3. https://docs.google.com で「空白のドキュメント」を作成
4. コピーした内容を貼り付け、体裁を整える
5. 「ファイル」→「移動」→ Spec Division フォルダに格納

**方法B: AIに格納方法を聞く**

1. Claude に「Google Docs を作成して Google Drive の特定フォルダに格納する方法」を聞く
2. 回答に従って実行する

**方法C: 他のやり方を探す**

1. AIに「Google Docs を自動作成する方法」「Google Drive に自動アップロードする方法」を聞いてみる
2. Google Apps Script、Zapier、その他のツールを使う方法もある
3. どんな手段でもいいので、手動でのファイル作成・アップロードを避ける方法を探る

## ヒント

- Claude の Web アプリとデスクトップアプリはどちらでも使える。ただし **デスクトップアプリは必ずダウンロードしておくこと**（今後の研修で使用する）
- ダウンロード: https://claude.ai/download
- 「AIにAI自身のことを聞く」のは有効なリサーチ手法。ただし自社製品に関する回答はバイアスがかかることがあるので注意
- リサーチ結果は「AIの出力をそのまま貼る」のではなく、自分の理解を加えて整理すること
- **手動で作成、格納しなければどんな手順でもOK** — いろいろ試してみること自体が学び

### 追加1: Claude のモデル比較

Claude のモデル（Opus, Sonnet, Haiku）について調べ、以下をまとめる:

**拡張思考の ON/OFF 方法:**

- 「Extended Thinking」のトグルを切り替える
